# DiskSentinel

> **当前版本：v1.0.0** | 仅支持 Windows 10/11

Windows C 盘文件监控与清理工具，本地运行的 Web 应用。通过浏览器直观查看磁盘使用状况、追踪文件变化趋势、安全清理无用文件。

## 功能概览

| 页面 | 功能 |
|------|------|
| **仪表盘** | 磁盘使用率、30 天趋势图、文件类型分布、最大目录 Top 100 |
| **快照管理** | 创建文件快照，完整记录当前 C 盘所有文件 |
| **快照对比** | 两次快照 diff，查看新增/删除/变大/变小的文件 |
| **清理中心** | 目录清理 + 规则清理双模式，备份后安全清理，支持一键恢复 |
| **实时监控** | 实时监控 C 盘文件变更，自动分析活跃目录，异常空间增长预警 |

## 启动方式

> **注意：本工具仅支持 Windows 10/11**，依赖 Windows 专有 API（`ReadDirectoryChangesW`、`GetDiskFreeSpaceExW` 等），不支持 macOS 和 Linux。

### 前置条件

- Windows 10/11
- Python 3.8+
- 管理员权限（扫描系统目录需要）

### 安装依赖

```bash
pip install fastapi uvicorn aiofiles sse-starlette pywin32
```

### 启动服务

```bash
cd disk-sentinel
python main.py
```

或双击 `启动.bat`（自动以管理员权限启动并打开浏览器）。

浏览器访问 http://localhost:8765

## 功能详解

### 仪表盘

进入页面自动扫描 C 盘，展示：

- **统计卡片**：总容量、已使用、可用空间、簇大小、使用率
- **使用趋势**：近 30 天磁盘使用量折线图
- **文件类型分布**：按分类（系统文件、可执行文件、图片、视频、文档等）统计占比饼图
- **最大目录 Top 100**：按含子目录总大小排序，自动折叠父子关系（子目录占父目录 90%+ 时隐藏父目录）

可勾选目录发送到清理中心。

### 快照管理

- 创建快照：遍历 C 盘所有文件，记录路径、大小、分配大小、修改时间、扩展名
- 后台线程扫描，SSE 实时推送进度
- 支持删除快照

### 快照对比

选择两个快照进行 diff，查看：

- **新增文件**：对比快照中有但基准快照中没有的文件
- **删除文件**：基准快照中有但对比快照中没有的文件
- **变大文件**：两个快照中都存在但大小增加的文件
- **变小文件**：两个快照中都存在但大小减少的文件

### 清理中心

两种清理模式：

**目录清理**：从仪表盘 Top 100 勾选目录，发送到清理中心后预览执行。

**规则清理**：内置 10 条清理规则，按分类管理：

| 分类 | 规则 |
|------|------|
| 临时文件 | 用户临时目录、系统临时目录、Prefetch |
| 缓存 | Chrome 缓存、Edge 缓存、缩略图缓存 |
| 日志 | Windows 错误报告、Windows 日志 |
| 系统更新 | Windows Update 下载缓存 |

每条规则可独立开关，支持按文件年龄过滤。

**安全机制**：

1. **预览**：执行前先查看将被清理的文件列表
2. **备份**：文件移动（而非删除）到指定磁盘，保留原始目录结构
3. **空间检查**：备份空间不足时拒绝执行
4. **失败回滚**：备份中途失败，已移动文件自动移回原位
5. **一键恢复**：从备份历史中随时恢复文件到 C 盘原位

系统关键目录（Windows、Program Files 等）硬编码保护，不可删除。

### 实时监控

基于 Windows `ReadDirectoryChangesW` API，实时监控 C 盘文件变更，通过 SSE 推送到前端。适合：

- 安装软件时观察文件写入位置
- 排查磁盘空间异常增长原因
- 监控后台程序的文件操作

**监控分析**：自动按目录聚合统计文件操作（新增/修改/删除次数、新增占用空间），展示活跃目录 Top 10。单目录新增占用超过 500MB 时标红警示，帮助快速定位空间异常增长的来源。

## 注意事项

清理功能请谨慎使用。虽然工具设计了备份与恢复机制，但仍存在以下风险：

- 某些程序运行时产生的临时文件被清理后，可能导致程序异常
- 部分缓存被清理后，相关软件需要重新下载或重建索引
- 规则清理覆盖的是通用场景，特殊软件的缓存路径可能未覆盖

**使用建议：**

- 清理前确认备份目标盘空间充足
- 不确定的目录先只预览、不执行，确认后再操作
- 重要数据提前自行备份，不要完全依赖工具的备份恢复功能

## 技术实现

- **后端**：Python + FastAPI + SQLite（WAL 模式）
- **前端**：纯 HTML/CSS/JS 单页应用 + Chart.js
- **实时推送**：SSE（sse-starlette）
- **磁盘信息**：ctypes 调用 `GetDiskFreeSpaceExW`、`GetDiskFreeSpaceW`
- **文件监控**：pywin32 `ReadDirectoryChangesW`
- **扫描引擎**：os.walk 一次遍历，同时收集目录大小和文件类型统计
- **并发模型**：后台线程执行耗时扫描，FastAPI 事件循环不阻塞
